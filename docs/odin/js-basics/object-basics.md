## Различия между объектами и примитивами

Ранее в курсе вы изучали примитивные типы данных. Теперь, когда вы познакомились с объектными типами данных (к которым относятся, в частности, объекты `{ключ: значение}`, массивы и функции), самое время понять основное различие между ними. Примитивы могут хранить только одно значение (строку, число и т.д.), тогда как объектные типы данных используются для хранения коллекций данных и более сложных сущностей.

Помимо формальных различий, существуют и технические отличия, влияющие на то, как мы работаем с каждым типом данных. Когда вы объявляете переменную примитивного типа, она содержит копию переданной информации:

```javascript
let data = 42;
// dataCopy будет хранить копию значения из data, то есть копию 42
let dataCopy = data;

// изменение dataCopy не повлияет на data
dataCopy = 43;

console.log(data); // 42
console.log(dataCopy); // 43
```

С другой стороны, когда вы объявляете переменную объектного типа, она содержит не само значение, а **ссылку** на объект:

```javascript
// obj содержит ссылку на объект, определённый справа
const obj = { data: 42 };
// objCopy будет содержать ту же ссылку, что и obj
const objCopy = obj;

// изменение objCopy повлияет на сам объект
objCopy.data = 43;

console.log(obj); // { data: 43 }
console.log(objCopy); // { data: 43 }
```

Это поведение вам уже знакомо. В предыдущем проекте вы изменяли ячейки сетки в Etch-A-Sketch, используя ссылки. Рассмотрим следующий пример кода:

```javascript
const element = document.querySelector("#container");
element.style.backgroundColor = "red";
```

Мы изменяем переменную `element`, но изменения применяются к соответствующему узлу в DOM. Почему так происходит? Потому что узел в нашем коде — это ссылка на тот же узел, который используется в DOM. Если бы это была не ссылка, а копия (как в случае с примитивами), изменения не оказали бы никакого эффекта, поскольку они применялись бы только к локальной копии.

Это поведение важно учитывать и при передаче аргументов в функцию. Рассмотрим следующие функции:

```javascript
function increaseCounterObject(objectCounter) {
  objectCounter.counter += 1;
}

function increaseCounterPrimitive(primitiveCounter) {
  primitiveCounter += 1;
}

const object = { counter: 0 };
let primitive = 0;

increaseCounterObject(object);
increaseCounterPrimitive(primitive);
```

Задумайтесь на мгновение и попробуйте предсказать, что произойдёт с `object` и `primitive` после вызова этих функций.

Если вы предположили, что значение счётчика в объекте увеличится на 1, а значение примитива останется неизменным, — вы правы. Дело в том, что параметр `objectCounter` содержит ссылку на тот же объект, что и переменная `object`, тогда как `primitiveCounter` получает лишь копию примитивного значения.

**Переназначение переменных объектного типа**

Хотя изменение (мутирование) объекта по ссылке затрагивает все переменные, ссылающиеся на этот объект, переназначение переменной не влияет на то, на что ссылаются другие переменные. Например:

```javascript
let animal = { species: "dog" };
let dog = animal;

// переназначаем переменную animal новым объектом
animal = { species: "cat" };

console.log(animal); // { species: "cat" }
console.log(dog); // { species: "dog" }
```

Таким образом, `dog` по-прежнему ссылается на исходный объект, несмотря на то, что `animal` теперь указывает на другой объект.

**В чём разница между объектами и массивами?**

 <details>  
 <summary>Ответ</summary>

Объекты — это коллекции данных, организованные в виде пар «ключ-значение», где каждый ключ имеет имя (строку или символ), а значение может быть любого типа.

Массивы — это особый вид объектов, предназначенный для хранения упорядоченных коллекций данных, доступ к которым осуществляется по числовым индексам (начиная с 0). Массивы используются, когда важен порядок элементов и требуется легко добавлять/удалять/получать элементы по индексу.

</details>

**Как получить доступ к свойствам объекта?**

 <details>  
 <summary>Ответ</summary>

Доступ к свойствам объекта можно получить двумя способами:

1. С помощью точечной нотации: `объект.ключ`
2. С помощью скобочной нотации: `объект["ключ"]`

Скобочная нотация полезна, когда имя свойства содержит пробелы, специальные символы или хранится в переменной.

</details>

**В чём разница между примитивами и объектными типами при их присваивании другим переменным или передаче в функции?**

 <details>  
 <summary>Ответ</summary>

При присваивании примитивных типов (например, строки, числа, булевы значения) новой переменной или передаче в функцию копируется само значение. Изменения в копии не влияют на оригинал.
При присваивании объектных типов (объекты, массивы, функции) передаётся не сам объект, а ссылка на него. Это означает, что если несколько переменных ссылаются на один и тот же объект, изменения через одну переменную будут видны через все остальные.

</details>
